# Learning Git — A Teacher's Log

I'm a CS teacher learning Git from scratch before teaching it 
in the fall. This repo is my practice space.

## Why I'm doing this
I'm teaching Software Studio in fall 2026 — a capstone course 
on systems-level software engineering. Git is foundational to 
the course, and I wanted real experience before asking 
students to use it.

## What I did today (May 7, 2026)
- Installed Git on a school Windows machine
- Made a note about ONE default to change when we install in class
- Configured my name and email
- Created this repository
- Made my first commits
- Created and merged a branch
- Made a note to introduce Git first through tree structure
- Pushed to GitHub for the first time--so cool
- Discovered VS Code's built-in Git interface

## What surprised me
- Branches are just pointers, not copies--extremely lightweight
- The commit history is a linked list; reconstruction is recursive tree traversal
- Git's internal elegance comes directly from CS concepts I learned in college almost 40 years ago
  
## What I still need to learn
- Pull and fetch
- Merge conflicts
- Pull request workflow

## Commands reference
- `git init` — create a repository
- `git status` — see what's changed
- `git add` — stage files
- `git commit -m "message"` — create a checkpoint
- `git push` — send commits to GitHub
